# IX-03 编码转换交互流程

> 对应页面：`03-encoding.html`（Hex）、`03b-base64.html`（Base64）、`03c-ascii.html`（ASCII）、`03d-url.html`（URL）、`03e-unicode.html`（Unicode）
> 覆盖范围：5 个编码/解码工具，共享通用交互模式

---

## 一、界面状态定义

### 1.1 通用状态（所有编码工具共享）

| 状态名称 | 触发条件 | 界面表现 |
|---------|---------|---------|
| 编码态 | 默认进入 / 点击"编码"标签 | 标签栏"编码"高亮，输入区标签为原始文本含义，输出区标签为编码结果含义 |
| 解码态 | 点击"解码"标签 | 标签栏"解码"高亮，输入区标签为编码文本含义，输出区标签为原始文本含义 |
| 转换完成态 | 输入内容变化后自动转换完成 | 输出区显示转换结果，信息提示区显示附加信息 |
| 错误态 | 输入内容无法转换 | 输出区显示错误提示（红色文字），说明错误原因 |
| 历史展开态 | 点击历史按钮 | 显示最近 5 条历史记录列表 |

### 1.2 各工具特有配置项

**Hex 转换（03-encoding.html）**

| 配置项 | 选项 | 默认值 |
|--------|------|--------|
| 分隔符 | 空格 / 逗号 / 无 | 空格 |
| 前缀 | 无 / `0x` / `\x` | 无 |
| 大小写 | 大写 / 小写 | 大写 |

额外按钮："紧凑复制"（复制时去除分隔符）

编码方式：使用 `TextEncoder` / `TextDecoder` 进行 UTF-8 编解码。

**Base64 编解码（03b-base64.html）**

| 配置项 | 选项 | 默认值 |
|--------|------|--------|
| 字符集 | UTF-8 / GBK / GB2312 / ISO-8859-1 | UTF-8 |
| 换行 | 无换行 / 每 64 字符 / 每 76 字符 | 无换行 |

编码方式：使用 `btoa` / `atob` 配合 `TextEncoder`。

**ASCII 转换（03c-ascii.html）**

| 配置项 | 选项 | 默认值 |
|--------|------|--------|
| 进制 | 十进制 / 十六进制 / 八进制 / 二进制 | 十进制 |
| 分隔符 | 空格 / 逗号 / 无 | 空格 |

额外元素：可折叠的 ASCII 参考表（展示 0-127 的字符对照）

非 ASCII 字符（码值 > 127）处理：标记为 `[非ASCII:code]` 格式。

**URL 编解码（03d-url.html）**

| 配置项 | 选项 | 默认值 | 可用模式 |
|--------|------|--------|---------|
| 编码模式 | 完整编码 / 组件编码 | 组件编码 | 仅编码态 |

- 完整编码：使用 `encodeURI`（保留 URL 结构字符 `:/?#[]@` 等）
- 组件编码：使用 `encodeURIComponent`（编码所有特殊字符）

信息提示区：显示编码后的体积膨胀百分比（如"编码后体积增加 35%"）。

**Unicode 编解码（03e-unicode.html）**

| 配置项 | 选项 | 默认值 |
|--------|------|--------|
| 格式 | `\uXXXX` / `U+XXXX` / HTML Entity / `&#XXXX;` | `\uXXXX` |
| 分隔符 | 空格 / 无 | 空格 |

特殊处理：对于码值 > `0xFFFF` 的字符（如 Emoji），使用代理对（surrogate pair）编码为两个 `\u` 序列。

### 1.3 通用界面元素

| 元素 | 说明 |
|------|------|
| 标签栏 | "编码" / "解码" 两个标签 |
| 选项栏 | 根据当前工具显示对应配置项 |
| 输入编辑器 | 带行号的文本编辑区域 |
| 交换按钮 | 输入区和输出区之间的双向箭头按钮 |
| 输出编辑器 | 显示转换结果的只读文本区域（带行号） |
| 信息提示区 | 显示附加信息（如字节数、膨胀率等） |
| 复制按钮 | 复制输出内容到剪贴板 |
| 清除按钮 | 清空输入和输出 |
| 历史按钮 | 展开/折叠历史记录列表 |

## 二、操作流程

### 2.1 标签切换流程（编码 ↔ 解码）

1. 用户点击标签 → 首先判断点击的标签是否与当前模式相同
2. **若点击的是当前已激活的标签（重复点击）**：
   - 不执行任何操作（不切换模式、不互换输入输出、不重新转换）
   - 忽略此次点击事件
3. **若点击的是另一个标签（实际切换）** → 以 编码→解码 为例：
   - 标签栏高亮切换到新标签
   - 若输出区有内容（且非错误提示），将输出区内容移入输入区
   - 更新输入区和输出区的标签文字（互换含义）
   - 使用新的编码方向对输入内容重新转换
   - **不清空**任何已有内容
4. 解码→编码 方向切换流程同上，反向操作

### 2.2 实时转换流程

1. 用户在输入区输入或修改文本
2. 每次输入变化（`input` 事件）立即触发转换：
   - 读取当前输入文本
   - 读取当前选项配置
   - 根据当前模式（编码/解码）执行对应转换
   - 将结果显示在输出区
3. 输入为空时 → 输出区清空
4. 转换失败时 → 进入 **错误态**，输出区显示红色错误提示

### 2.3 交换按钮流程

1. 用户点击交换按钮（双向箭头图标）
2. 将输出区当前内容写入输入区
3. 切换当前模式（编码 ↔ 解码），更新标签高亮和标签文字
4. 使用新模式对输入内容重新执行转换
5. 更新输出区显示新的转换结果

### 2.4 选项变更流程

1. 用户修改任意配置项（分隔符、前缀、大小写等）
2. 立即使用新配置对当前输入重新执行转换
3. 输出区更新为新的转换结果
4. 不影响输入区内容

### 2.5 复制按钮流程

1. 用户点击"复制"按钮
2. 将输出区文本写入系统剪贴板
3. 按钮文字变为"已复制"，文字颜色变为绿色
4. 持续 1.2 秒后恢复为"复制"按钮的原始状态
5. 输出区为空时点击复制按钮 → 不执行复制，无反馈

**Hex 工具额外行为**：
- "紧凑复制"按钮：复制时去除所有分隔符，仅保留纯 Hex 字符串

### 2.6 历史记录流程

1. 每次成功转换后，将一条记录添加到历史列表：
   - 记录内容：截断后的输入文本（前 30 字符）+ 截断后的输出文本（前 30 字符）+ 当前模式 + 时间戳
2. 历史列表最多保留 5 条记录
3. 超过 5 条时，最早的记录被移除（FIFO）
4. 用户点击历史按钮 → 展开/折叠历史记录列表
5. 用户点击某条历史记录 → 恢复该记录的输入内容到输入区，恢复模式，重新执行转换

### 2.7 清除按钮流程

1. 用户点击"清除"按钮
2. 输入区文本清空
3. 输出区文本清空
4. 信息提示区清空
5. 不清除历史记录
6. 不重置选项配置

### 2.8 ASCII 工具特有流程：参考表折叠

1. 页面底部默认显示折叠状态的"ASCII 参考表"
2. 用户点击标题 → 展开表格，显示 0-127 的字符对照（十进制、十六进制、字符）
3. 再次点击 → 折叠表格
4. 参考表不影响输入输出区域

## 三、交互规则

### 3.1 实时转换规则

| 规则 | 说明 |
|------|------|
| 触发时机 | `input` 事件，即每次输入变化都触发转换 |
| 防抖 | 无需防抖，编码转换均为同步轻量计算，可直接执行 |
| 空输入 | 输入为空时输出清空，不显示错误 |
| 错误处理 | 转换失败时输出区显示具体错误原因（如"输入不是有效的 Base64 字符串"） |

### 3.2 标签切换规则

1. 点击标签前先判断：若点击的标签与当前激活标签相同，**直接忽略**，不做任何状态变更或内容移动
2. 仅在实际切换模式（编码↔解码）时才执行以下逻辑
3. 实际切换时 **不清空**输入和输出内容
4. 若输出区有内容（且非错误提示），切换时将其移入输入区作为新输入
5. 切换后立即对新输入执行反向转换
6. 若输出区为空，切换后输入区保持为空

### 3.3 交换按钮规则

1. 交换操作等同于：复制输出 → 粘贴到输入 → 切换模式 → 重新转换
2. 输出区为空时点击交换 → 仅切换模式，输入区清空
3. 交换按钮始终可见且可点击（非禁用状态）

### 3.4 复制反馈规则

1. 复制成功后按钮文字变为"已复制"，颜色变为绿色
2. 反馈持续 1.2 秒后自动恢复
3. 反馈期间按钮仍可点击（不会重复反馈）
4. 使用 `navigator.clipboard.writeText()` 或 Tauri 剪贴板 API

### 3.5 历史记录规则

| 规则 | 说明 |
|------|------|
| 最大条数 | 5 条 |
| 存储方式 | 会话内存储（页面级变量），不持久化到 localStorage |
| 记录内容 | 截断输入（前 30 字符）→ 截断输出（前 30 字符） |
| 点击恢复 | 恢复输入内容和模式，触发重新转换 |
| 清除影响 | 点击"清除"按钮不清除历史 |

### 3.6 错误处理规则

1. 转换失败时输出区文字变为红色
2. 错误提示需说明具体原因，而非通用错误文案
3. 常见错误场景：
   - Hex 解码：输入包含非 Hex 字符
   - Base64 解码：输入不是有效 Base64 格式
   - ASCII 解码：输入包含非数字分隔符以外的字符
   - URL 解码：输入包含无效的 `%` 编码序列
   - Unicode 解码：输入包含无效的 Unicode 转义序列
4. 错误状态下仍可继续编辑输入，修改后自动重新尝试转换

### 3.7 各工具特殊规则

**Hex 转换**
- 编码使用 `TextEncoder` 将字符串转为 UTF-8 字节数组，再逐字节转 Hex
- 解码使用 `TextDecoder` 将 Hex 字节数组还原为字符串
- "紧凑复制"仅去除分隔符，不改变前缀和大小写

**Base64 编解码**
- 非 UTF-8 字符集（GBK/GB2312）需使用对应编码库
- 换行选项仅影响输出显示格式，不影响复制内容

**ASCII 转换**
- 非 ASCII 字符（码值 > 127）标记为 `[非ASCII:code]`，其中 `code` 为实际 Unicode 码值
- 解码时遇到 `[非ASCII:code]` 标记，还原为对应 Unicode 字符

**URL 编解码**
- "完整编码"使用 `encodeURI`，保留 `;/?:@&=+$,#` 等 URL 结构字符
- "组件编码"使用 `encodeURIComponent`，编码所有非字母数字字符
- 信息提示区显示：原始字节数 → 编码后字节数 → 膨胀百分比

**Unicode 编解码**
- 代理对处理：码值 > `0xFFFF` 的字符拆分为高低代理对，输出为两个连续的 `\u` 序列
- 解码时正确识别并合并代理对
- HTML Entity 格式使用 `&#xXXXX;` 形式

### 3.8 配置选项去重规则

1. 所有配置下拉框/选择器（分隔符、前缀、大小写、字符集、换行、进制、编码模式、格式等）**重复选择当前已选中的选项则忽略**，不重复触发转换
2. 适用范围：Hex（分隔符、前缀、大小写）、Base64（字符集、换行）、ASCII（进制、分隔符）、URL（编码模式）、Unicode（格式、分隔符）
