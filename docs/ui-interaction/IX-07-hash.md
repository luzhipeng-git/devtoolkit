# IX-07 哈希摘要交互流程

> 涵盖页面：`07-hash.html`

---

## 一、界面状态定义

| 状态名称 | 说明 | 界面表现 |
|---------|------|---------|
| 初始态 | 页面加载完成 | 算法选择器默认选中 SHA-256；HMAC 区域隐藏（非 HMAC 算法）；输入编辑器为空；结果卡片显示占位文本「请输入内容并点击计算」；其他算法结果区折叠；历史记录区为空 |
| HMAC 模式态 | 选择了 HMAC-* 算法 | HMAC Secret Key 输入区域显示在算法选择器下方；Secret Key 输入框为空，带眼睛图标切换可见性；输入编辑器无变化 |
| 非 HMAC 模式态 | 选择了非 HMAC 算法 | HMAC Secret Key 区域隐藏；Secret Key 值不丢失（切换回 HMAC 时恢复） |
| 输入态 | 用户正在输入编辑器中输入内容 | 输入编辑器获得焦点，边框高亮；结果卡片保持上次结果或占位文本；字数统计实时更新 |
| 计算中态 | 用户点击「计算」按钮后 | 「计算」按钮显示 loading spinner 并短暂禁用（通常 < 100ms）；输入编辑器保持不变 |
| 结果态 | 哈希计算完成 | 主结果卡片显示：算法名称标签 + 字符数徽章 + 哈希值（等宽字体显示）+ 复制按钮；其他算法结果已在后台完成计算（折叠状态） |
| 展开态 | 用户点击「查看其他算法结果」 | 其他算法结果区域展开，以网格形式显示所有其他算法的哈希值；展开箭头旋转 180 度 |
| 折叠态 | 用户再次点击收起 | 其他算法结果区域折叠隐藏；展开箭头恢复向上 |
| 复制反馈态 | 用户点击复制按钮 | 对应复制按钮图标变为勾选，文字变为「已复制」；2 秒后恢复 |
| 异常态 | 计算失败（如 HMAC 缺少密钥） | 输入区域上方显示红色错误横幅；结果卡片保持上次结果或占位文本 |

## 二、操作流程

### 2.1 哈希计算主流程

1. 用户从算法选择器中选择目标算法（MD5 / SHA-1 / SHA-256 / SHA-384 / SHA-512 / HMAC-SHA256 / HMAC-SHA384 / HMAC-SHA512 / CRC32）
2. 若选择 HMAC 系列算法，HMAC Secret Key 输入区域自动显示
   - 用户需在 Secret Key 输入框中输入密钥
   - 可点击眼睛图标切换密钥显示/隐藏
3. 用户在输入编辑器（contenteditable div）中输入或粘贴待计算的内容
4. 输入编辑器右上角实时显示字符数统计
5. 用户点击「计算」按钮
6. 系统执行以下操作：
   - 使用 Web Crypto API 计算所选算法的哈希值（SHA 系列）
   - 使用内置实现计算 MD5 和 CRC32
   - HMAC 系列使用 Web Crypto API 的 HMAC 模式
   - 同时在后台计算所有其他算法的哈希值
7. 主结果卡片更新为当前算法的结果
8. 其他算法结果缓存到内存中（不立即显示）

### 2.2 查看其他算法结果流程

1. 用户点击「查看其他算法结果」区域（折叠条/展开条）
2. 其他算法结果区域以动画展开（向下滑动 + 箭头旋转 180 度）
3. 以网格形式展示所有其他算法的哈希值，每个算法卡片包含：
   - 算法名称
   - 哈希值（等宽字体，可选中复制）
   - 独立复制按钮
4. 用户可再次点击折叠条收起结果区域

### 2.3 算法切换流程

1. 用户在算法选择器中切换到不同的算法
2. 界面根据新算法类型显示或隐藏 HMAC 区域
3. 主结果卡片头部更新为新算法名称
4. 若已有计算结果：
   - 若后台已缓存该算法结果，直接显示
   - 若尚未计算，结果区域显示「点击计算获取结果」
5. 输入内容和 Secret Key 保持不变

### 2.4 历史记录流程

1. 每次成功计算后，系统自动在历史记录区新增一条记录
2. 历史记录条目包含：计算时间、所选算法、输入内容摘要（前 30 字符）、哈希值前 16 字符
3. 历史记录最多保留 5 条，新记录插入顶部，超出时移除最旧的记录
4. 用户点击某条历史记录，可将该记录的输入内容填入输入编辑器并切换到对应算法

## 三、交互规则

| 规则编号 | 触发条件 | 行为描述 | 去重保护 | 约束说明 |
|---------|---------|---------|---------|---------|
| R-07-01 | 算法选择器变更 | 更新结果卡片标题为新算法名；显示/隐藏 HMAC Secret Key 区域；若已有缓存结果则直接显示，否则显示占位文本 | **重复选择当前已选中的算法则忽略**，不重复显示/隐藏 HMAC 区域 | 切换算法不清空输入内容 |
| R-07-02 | 选择 HMAC-* 算法 | 在算法选择器下方平滑展开 HMAC Secret Key 输入区域；Secret Key 输入框获得焦点 | —（R-07-01 的子状态） | 使用 CSS transition 实现展开动画 |
| R-07-03 | 从 HMAC-* 切换到非 HMAC 算法 | HMAC Secret Key 输入区域平滑收起隐藏；已输入的 Secret Key 值保留在内存中 | —（R-07-01 的子状态） | 切换回 HMAC 时恢复之前的密钥值 |
| R-07-04 | 点击「计算」按钮 | 校验：输入内容非空；HMAC 模式时 Secret Key 非空；校验通过执行计算；校验失败在对应输入框下方显示错误信息 | 无需去重（操作按钮有明确意图） | HMAC Secret Key 为空时提示「HMAC 计算需要密钥」 |
| R-07-05 | 哈希计算执行 | 主算法计算结果显示在主结果卡片；所有其他算法同时计算并缓存；按钮短暂禁用后恢复 | —（计算执行，不涉及选择型控件） | 所有计算在前端完成，无网络请求 |
| R-07-06 | 主结果卡片显示 | 卡片包含：算法名称（大字标题）+ 字符数徽章（如 "64字符"）+ 哈希值（等宽字体整行显示）+ 复制按钮 | —（展示规则，不涉及选择型控件） | 哈希值过长时自动换行，不允许水平滚动 |
| R-07-07 | 点击「查看其他算法结果」折叠条 | 展开时：区域向下滑动展开，箭头旋转 180 度，显示所有其他算法结果网格；折叠时：区域向上滑动收起，箭头恢复 | —（toggle 开关，不涉及选择型控件） | 使用 CSS transition，动画时长 300ms |
| R-07-08 | 其他算法结果网格 | 每个算法一个卡片，2 列网格布局；每张卡片显示算法名 + 哈希值 + 复制按钮 | —（展示规则，不涉及选择型控件） | 不包含当前选中的主算法 |
| R-07-09 | 点击复制按钮（主结果或其他算法） | 将对应哈希值复制到剪贴板；按钮图标变为勾选 + 「已复制」；2 秒后恢复原始图标 | —（操作按钮，不涉及选择型控件） | 使用 navigator.clipboard.writeText API |
| R-07-10 | 历史记录自动添加 | 每次成功计算后自动在历史区顶部新增一条记录；超过 5 条时移除最旧的一条 | —（自动行为，不涉及选择型控件） | 历史记录仅保存在当前会话，不持久化 |
| R-07-11 | 历史记录点击 | 将该记录的输入内容填入输入编辑器；将算法选择器切换为该记录的算法；不自动触发计算 | —（操作按钮，不涉及选择型控件） | 用户需手动点击「计算」查看结果 |
| R-07-12 | 输入编辑器实时字数统计 | 输入编辑器右上角显示当前字符数，如「字符数: 256」；用户输入时实时更新 | —（展示规则，不涉及选择型控件） | 统计使用 String.length，含空格和换行 |
| R-07-13 | Secret Key 可见性切换 | 点击眼睛图标在明文和圆点遮罩之间切换 | —（toggle 开关，不涉及选择型控件） | 使用 CSS text-security 属性 |
| R-07-14 | MD5 和 CRC32 计算 | 使用内置 JavaScript 实现，非 Web Crypto API；结果格式与 SHA 系列一致（Hex 字符串） | —（计算逻辑，不涉及选择型控件） | MD5 输出 32 字符，CRC32 输出 8 字符 |
| R-07-15 | 空输入点击计算 | 在输入编辑器下方显示提示「请输入待计算的内容」；不执行计算 | —（校验规则，不涉及选择型控件） | 输入框边框临时变红，1.5 秒后恢复 |
