# IX-08 密钥工具交互流程

> 涵盖页面：`08-keygen.html`

---

## 一、界面状态定义

| 状态名称 | 说明 | 界面表现 |
|---------|------|---------|
| 初始态 | 页面加载完成 | 密钥长度默认 32 字节；密钥格式默认 Hex；生成数量默认 1；密钥卡片区域为空；KCV 密钥输入为空；KCV 算法默认 AES；页面加载后自动生成一个密钥 |
| 待生成态 | 参数已设置，尚未点击生成 | 密钥长度/格式/数量选项已就绪；密钥卡片区域显示上次生成结果或为空 |
| 生成中态 | 用户点击「生成」按钮 | 「生成」按钮短暂显示 loading spinner（通常 < 50ms）；数量较多时可能有轻微延迟 |
| 生成结果态 | 密钥生成完成 | 密钥以卡片形式展示，每张卡片包含：密钥值 + 长度信息 + 单独复制按钮；若数量 > 1，显示「批量复制」按钮 |
| 批量复制态 | 数量 > 1 的生成结果 | 在所有卡片下方显示「批量复制」按钮；点击后复制所有密钥（换行分隔） |
| 单条复制反馈态 | 用户点击单张卡片的复制按钮 | 该卡片的复制按钮图标变为勾选 + 「已复制」；2 秒后恢复 |
| KCV 待计算态 | KCV 区域参数已填入 | 密钥输入框为空或已填入 Hex 密钥；算法已选择；KCV 输出区为空或显示占位文本 |
| KCV 计算中态 | 用户点击「计算 KCV」 | 按钮短暂显示 loading spinner |
| KCV 结果态 | KCV 计算完成 | KCV 输出区显示 6 个 Hex 字符 + 复制按钮 |
| KCV 异常态 | KCV 计算失败 | 密钥输入框下方显示红色错误信息（密钥格式错误、长度不匹配等） |

## 二、操作流程

### 2.1 随机密钥生成流程

1. 用户在「随机密钥生成」区域设置参数：
   - 密钥长度（字节数）：通过数字输入框或滑块设置，默认 32，范围 1-256
   - 密钥格式：Hex（默认）或 Base64 单选
   - 生成数量：数字输入框，默认 1，范围 1-100
2. 用户点击「生成」按钮
3. 系统使用 crypto.getRandomValues 生成指定数量的随机密钥
   - 每个密钥为指定长度的随机字节数组
   - 按 Hex 或 Base64 格式编码
4. 生成结果以卡片网格形式展示：
   - 每张卡片包含：密钥值（等宽字体）、长度信息（如「32 字节 / 64 Hex 字符」）、复制按钮
   - 卡片排列为响应式网格（桌面端多列，窄屏单列）
5. 若生成数量 > 1，在卡片区域下方显示「批量复制」按钮
6. 每次点击「生成」替换之前的所有结果

### 2.2 单条密钥复制流程

1. 用户点击某张密钥卡片右上角的复制按钮
2. 系统将该卡片的密钥值复制到剪贴板
3. 该卡片的复制按钮图标变为勾选 + 「已复制」文字
4. 2 秒后按钮恢复为原始复制图标

### 2.3 批量复制流程

1. 用户点击「批量复制」按钮（仅在生成数量 > 1 时可见）
2. 系统将所有密钥以换行符（\n）分隔拼接后复制到剪贴板
3. 按钮图标变为勾选 + 「已全部复制」
4. 2 秒后恢复

### 2.4 KCV 计算流程

1. 用户在「KCV 计算」区域的密钥输入框中输入 Hex 格式的密钥，支持以下输入方式：
   - **手动输入**：在输入框中直接输入或粘贴 Hex 格式密钥
   - **文件上传**：点击「上传文件」按钮，选择包含 Hex 密钥的文本文件（.txt / .key），系统自动读取内容并填入
2. 用户从算法下拉框中选择算法（AES / DES / 3DES）
3. 用户点击「计算 KCV」按钮
4. 系统校验密钥格式和长度：
   - AES：16/24/32 字节
   - DES：8 字节
   - 3DES：16 或 24 字节
5. 系统计算 KCV（密钥校验值）：
   - 使用密钥对全零块进行加密
   - 取加密结果的前 3 个字节（6 个 Hex 字符）
   - 注：当前版本使用 XOR 折叠模拟实现
6. KCV 输出区显示 6 个 Hex 字符
7. 用户可点击复制按钮复制 KCV 值

### 2.5 初始加载流程

1. 页面加载完成后，自动以默认参数（32 字节、Hex、数量 1）生成一个随机密钥
2. 密钥卡片立即显示，用户可直接复制使用
3. KCV 区域保持空状态，等待用户输入

## 三、交互规则

| 规则编号 | 触发条件 | 行为描述 | 去重保护 | 约束说明 |
|---------|---------|---------|---------|---------|
| R-08-01 | 页面初始加载 | 以默认参数自动生成 1 个 32 字节 Hex 密钥并显示 | —（页面加载，不涉及用户主动选择） | 确保用户打开页面即可看到可用密钥 |
| R-08-02 | 密钥长度变更 | 更新长度显示；不自动重新生成密钥；用户需手动点击「生成」 | —（数字输入框，不涉及选择型控件） | 范围限制 1-256 字节 |
| R-08-03 | 密钥格式切换（Hex↔Base64） | 若已有生成结果，将现有密钥值从旧格式重新编码为新格式显示 | **重复选择当前密钥格式则忽略**，不重复执行重新编码 | 不重新生成随机值，仅改变编码 |
| R-08-04 | 生成数量变更 | 更新数量显示；不自动重新生成 | —（数字输入框，不涉及选择型控件） | 范围限制 1-100 |
| R-08-05 | 点击「生成」按钮 | 使用 crypto.getRandomValues 生成指定数量的随机密钥；以卡片网格形式展示；替换之前的所有结果 | 无需去重（操作按钮有明确意图） | 生成是不可逆操作，旧结果不保留 |
| R-08-06 | 生成结果卡片布局 | 桌面端：每行 2-3 张卡片；平板：每行 2 张；手机：每行 1 张；卡片之间等间距 | —（展示规则，不涉及用户主动选择） | 使用 CSS Grid 响应式布局 |
| R-08-07 | 单张卡片复制 | 将该卡片密钥值复制到剪贴板；仅影响该卡片的复制按钮状态 | —（操作按钮，不涉及选择型控件） | 不影响其他卡片 |
| R-08-08 | 批量复制按钮显示 | 生成数量 = 1 时隐藏；生成数量 > 1 时显示在卡片区域下方 | —（展示规则，不涉及用户主动选择） | 按钮仅在多密钥时可见 |
| R-08-09 | 批量复制执行 | 将所有密钥以换行分隔拼接后复制；顺序与卡片显示顺序一致 | —（操作按钮，不涉及选择型控件） | 使用 \n 作为分隔符 |
| R-08-10 | KCV 密钥输入格式 | 仅接受 Hex 字符（0-9, a-f, A-F）；非 Hex 字符输入时显示红色边框 + 提示「仅支持 Hex 格式」 | —（输入校验，不涉及选择型控件） | 实时校验，输入时即反馈 |
| R-08-11 | KCV 密钥长度校验 | 点击「计算 KCV」时校验密钥长度是否符合所选算法要求；不符合时在输入框下方显示具体错误（如「AES 密钥长度应为 16/24/32 字节，当前 12 字节」） | —（校验规则，不涉及选择型控件） | 校验在点击计算时触发，非实时 |
| R-08-12 | KCV 计算结果 | 显示 6 个大写 Hex 字符（如 `A1B2C3`）；等宽字体显示；附带复制按钮 | —（展示规则，不涉及用户主动选择） | 固定 6 字符，不足时补前导零 |
| R-08-13 | KCV 算法切换 | 无即时效果；下次计算时使用新算法的长度校验规则 | **重复选择当前 KCV 算法则忽略** | 不清空已有的密钥输入和 KCV 结果 |
| R-08-14 | 密钥卡片信息展示 | 每张卡片显示：密钥值（等宽字体，长密钥自动换行）+ 「X 字节 / Y Hex 字符」或「X 字节 / Base64」信息标签 + 复制按钮 | —（展示规则，不涉及用户主动选择） | 密钥值可选中但不可编辑 |
| R-08-15 | 大数量生成（>50） | 生成数量较大时，卡片区域使用虚拟滚动优化渲染性能；生成按钮在全部完成前保持禁用 | —（展示规则，不涉及用户主动选择） | 100 条密钥生成通常 < 100ms |
| R-08-16 | KCV 密钥文件上传 | KCV 密钥输入框支持文件上传，点击「上传文件」按钮后打开系统文件选择对话框；支持的文件格式：.txt、.key；文件读取后自动填入输入框，自动去除空白字符和换行 | —（操作按钮，不涉及选择型控件） | 文件大小限制 1MB；文件内容仅保留 Hex 字符 |
