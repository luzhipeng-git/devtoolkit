# IX-04 JSON 工具交互流程

> 对应页面：`04-json.html`（格式化 & 压缩）、`04c-json-deserialize.html`（反序列化）、`04d-json-path.html`（JSONPath 查询）、`04e-json-diff.html`（JSON Diff）
> 覆盖范围：4 个 JSON 处理工具（Pretty 格式化与 JSON 压缩已合并为一个页面）

---

## 一、界面状态定义

### 1.1 通用状态（JSON 工具共享）

| 状态名称 | 触发条件 | 界面表现 |
|---------|---------|---------|
| 初始态 | 工具页面加载完成 | 输入区为空，输出区为空，操作按钮可用 |
| 待操作态 | 用户输入了内容但未点击操作按钮 | 输入区有内容，输出区为空或保持上次结果 |
| 结果态 | 用户点击操作按钮后成功处理 | 输出区显示处理结果（带语法高亮或差异标记） |
| 错误态 | 输入内容无效或处理失败 | 输出区或输入区下方显示红色错误提示 |

### 1.2 格式化 & 压缩（04-json.html）—— 合并页面

**设计思路**：输入一次 JSON，同时输出格式化和压缩两种结果，无需切换页面。

**配置选项**

| 配置项 | 选项 | 默认值 | 影响范围 |
|--------|------|--------|---------|
| 缩进方式 | 2 空格 / 4 空格 / Tab | 2 空格 | 仅格式化面板 |
| 排序 Keys | 开 / 关 | 关 | 格式化 + 压缩 |
| Unicode 转义 | 开 / 关 | 关 | 格式化 + 压缩 |

**界面布局**

| 区域 | 说明 |
|------|------|
| 输入区 | 多行文本编辑区域（textarea），用于粘贴原始 JSON |
| 按钮栏 | "格式化 & 压缩"主按钮（强调样式）+ "重置"按钮 |
| 输出区 - 格式化 | 左侧面板（全高）：语法高亮的格式化 JSON 展示区域，右上角"复制"按钮，下方显示字符数 |
| 配置区 | 右侧面板上半部分：缩进方式下拉、排序 Keys 开关、Unicode 转义开关。利用压缩输出区较矮的特点，与其等高并排 |
| 输出区 - 压缩 | 右侧面板下半部分（矮）：压缩为单行的 JSON 字符串，右上角"复制"按钮，下方显示字符数 |
| 历史区域 | 可展开的历史记录列表 |

**输出区双面板布局**

```
┌─────────────────────────────────────────────────┐
│  输入区（JSON）                        [清空]     │
│  {"name":"张三","age":25,"address":{...}}        │
├─────────────────────────────────────────────────┤
│           [ 格式化 & 压缩 ]    [ 重置 ]           │
├──────────────────────┬──────────────────────────┤
│                      │  配置区                    │

│              [复制]   │  │ 缩进: [2空格 ▾]       │ │
│  {                   │  │ ☐ 排序 Keys           │ │
│    "name": "张三",    │  │ ☐ Unicode 转义        │ │
│    "age": 25,         │  └──────────────────────┘ │
│    "address": {       ├──────────────────────────┤
│      "city": "北京"   │  压缩（Minified）          │
│    }                  │                [复制]      │
│  }                    │  {"name":"张三","age":25,  │
│                       │  "address":{"city":"北京",  │

│                       │  "skills":["Java","Python"]}│
│                       │  82 字符                    │
├──────────────────────┴──────────────────────────┤
│  压缩率: 51%（168 → 82 字符）                     │
├─────────────────────────────────────────────────┤
│  历史记录（最近 5 条）                   [清空]    │
└─────────────────────────────────────────────────┘
```

**语法高亮样式**

| JSON 元素 | CSS 类名 | 颜色说明 |
|-----------|----------|---------|
| 键名（Key） | `json-key` | 蓝色或紫色 |
| 字符串值（String） | `json-string` | 绿色 |
| 数字值（Number） | `json-number` | 橙色 |
| 布尔值（Boolean） | `json-bool` | 红色或粉色 |
| null 值 | `json-null` | 灰色 |
| 括号（Bracket） | `json-bracket` | 黑色或深灰 |

### 1.4 反序列化（04c-json-deserialize.html）

**配置选项**

| 配置项 | 选项 | 默认值 |
|--------|------|--------|
| 目标语言 | Java / Python / Go / TypeScript / C# / Rust | Java |

**行为**：将 JSON 字符串转换为指定语言的数据结构定义（如 Java 的 POJO 类、Python 的 dataclass、Go 的 struct 等）。

### 1.5 JSONPath 查询（04d-json-path.html）

**界面布局**

| 区域 | 说明 |
|------|------|
| JSON 输入区 | 输入待查询的 JSON 数据 |
| JSONPath 输入框 | 单行输入框，输入 JSONPath 表达式（如 `$.store.book[*].title`） |
| 查询按钮 | 执行 JSONPath 查询 |
| 结果区 | 显示匹配结果的列表或 JSON 片段 |
| 匹配数量 | 显示"找到 N 个匹配"的计数 |

### 1.6 JSON Diff（04e-json-diff.html）

**界面布局**

| 区域 | 说明 |
|------|------|
| 左侧输入区（JSON A） | 输入第一份 JSON |
| 右侧输入区（JSON B） | 输入第二份 JSON |
| 按钮栏 | "对比"主按钮 |
| 差异结果区 | 显示逐行对比结果，颜色标记差异 |
| 统计栏 | 显示差异统计：相同 / 新增 / 删除 / 修改的数量 |
| 图例 | 颜色含义说明 |

**差异颜色标记**

| 差异类型 | 颜色 | 含义 |
|---------|------|------|
| ADD | 绿色 | JSON B 中新增的内容 |
| DEL | 红色 | JSON B 中删除的内容 |
| MOD | 黄色 | 两个 JSON 中值不同的字段 |
| 相同 | 默认色 | 两份 JSON 中一致的内容 |

**统计栏格式**

```
相同: 12  |  新增: 3  |  删除: 1  |  修改: 4
```

## 二、操作流程

### 2.1 格式化 & 压缩流程（合并页面）

1. 用户在输入区粘贴或输入 JSON 字符串
2. 用户根据需要调整选项（缩进方式、排序 Keys、Unicode 转义）
3. 用户点击"格式化 & 压缩"按钮
4. 系统解析输入 JSON：
   - 解析失败 → 进入 **错误态**，在输入区下方显示错误提示（包含行号和错误原因），两个输出区均清空
   - 解析成功 → 同时生成两种输出：
     - **格式化面板**：按选项配置格式化 JSON，带语法高亮显示
     - **压缩面板**：去除所有空白和换行，压缩为单行 JSON 字符串
   - 底部显示压缩率信息（如"压缩率: 51%（168 → 82 字符）"）
5. 两个输出面板各自支持独立复制

**重置按钮流程**：
1. 用户点击"重置"按钮
2. 清空输入区和两个输出区
3. 不重置选项配置

### 2.3 反序列化流程

1. 用户在输入区粘贴 JSON 字符串
2. 用户选择目标语言（Java / Python / Go / TypeScript / C# / Rust）
3. 用户点击"生成"按钮
4. 系统解析 JSON 结构并生成目标语言的代码：
   - 成功 → 输出区显示生成的代码（带语法高亮）
   - 失败 → 显示错误提示
5. 输出区代码可复制

### 2.4 JSONPath 查询流程

1. 用户在 JSON 输入区输入待查询的 JSON 数据
2. 用户在 JSONPath 输入框输入查询表达式（如 `$.store.book[*].author`）
3. 用户点击"查询"按钮
4. 系统执行 JSONPath 查询：
   - JSON 无效 → 显示 JSON 解析错误
   - JSONPath 表达式无效 → 显示表达式错误提示
   - 查询成功 → 结果区显示匹配项列表，统计区显示"找到 N 个匹配"
   - 无匹配 → 结果区显示"无匹配结果"
5. 用户修改表达式或 JSON 后，需再次点击"查询"按钮执行

### 2.5 JSON Diff 流程

1. 用户在左侧输入区（JSON A）输入第一份 JSON
2. 用户在右侧输入区（JSON B）输入第二份 JSON
3. 用户点击"对比"按钮
4. 系统执行差异比较：
   - 任一 JSON 无效 → 显示对应输入区的解析错误
   - 比较成功 → 差异结果区显示逐行对比结果：
     - 相同行以默认色显示
     - 仅在 JSON A 中存在的行以红色（DEL）标记
     - 仅在 JSON B 中存在的行以绿色（ADD）标记
     - 两边都存在但值不同的行以黄色（MOD）标记
   - 统计栏更新：显示相同 / 新增 / 删除 / 修改的计数
5. 图例始终显示在差异结果区上方或下方

### 2.6 复制功能流程（所有 JSON 工具通用）

1. 输出区旁显示"复制"按钮
2. 用户点击 → 输出内容写入剪贴板
3. 按钮文字变为"已复制"，颜色变绿
4. 1.2 秒后恢复原状

### 2.7 历史记录流程（格式化 & 压缩工具）

1. 每次成功执行格式化或压缩操作后，记录一条历史
2. 记录内容：截断后的输入（前 50 字符）+ 操作类型 + 时间戳
3. 最多保留 5 条历史记录
4. 用户点击历史条目 → 输入区恢复该记录的输入内容
5. 不自动执行操作（需用户再次点击按钮）

## 三、交互规则

### 3.1 显式操作按钮规则（与编码工具的核心区别）

| 规则 | 说明 |
|------|------|
| 非实时转换 | 输入变化 **不自动触发** 处理，必须用户点击操作按钮 |
| 按钮类型 | 各工具的主操作按钮：格式化 & 压缩 / 生成 / 查询 / 对比 |
| 按钮样式 | 主按钮使用强调色（如蓝色或主题色），与重置按钮视觉区分 |
| 快捷键支持 | 可选支持 `Ctrl+Enter` 作为执行操作的快捷键 |

### 3.2 JSON 解析错误处理规则

1. 解析失败时，尽可能提供精确的错误位置信息：
   - 行号和列号（如"第 3 行第 12 列"）
   - 错误原因（如"缺少右括号"、"意外的 token"）
2. 错误提示显示在输入区下方，红色文字
3. 错误提示不覆盖输入区内容
4. 用户修改输入后，错误提示不清除，直到再次点击操作按钮
5. 常见错误场景：
   - 未闭合的字符串或括号
   - 尾随逗号
   - 注释（标准 JSON 不支持注释）
   - 单引号代替双引号

### 3.3 格式化 & 压缩规则（合并页面）

| 规则 | 说明 |
|------|------|
| 一次操作双输出 | 点击"格式化 & 压缩"按钮同时生成格式化和压缩两种结果，无需分别操作 |
| 配置区位置 | 配置区位于右侧面板上半部分，与压缩输出区等高并排，充分利用压缩输出较矮的空间 |
| 格式化面板 | 左侧全高面板，按缩进方式配置格式化，带语法高亮（`<pre><code>` + CSS 类），显示字符数 |
| 压缩面板 | 右侧下半面板（矮），去除所有空白和换行，输出单行 JSON 字符串，无语法高亮，显示字符数 |
| 压缩率 | 底部信息栏显示压缩率百分比和原始→压缩字符数对比 |
| 独立复制 | 两个输出面板各有独立的"复制"按钮，互不影响 |
| 缩进方式 | 2 空格 / 4 空格 / Tab 三种选择，**仅影响格式化面板的缩进**，对压缩输出无效 |
| 排序 Keys | 开启后按字母顺序对所有对象的键名排序（递归），同时影响格式化和压缩输出 |
| Unicode 转义 | 开启后将非 ASCII 字符转义为 `\uXXXX` 格式，同时影响格式化和压缩输出 |
| 选项去重 | 缩进方式、排序 Keys、Unicode 转义等配置项，**重复选择当前已选中的选项则忽略**，不重复触发格式化或压缩 |
| 选项变更 | 修改选项后需重新点击按钮才能更新输出，不自动触发 |

### 3.4 JSON Diff 布局与对比规则

1. 使用左右并排布局（side-by-side），而非上下布局
2. 左侧输入区和右侧输入区宽度相等
3. 差异结果区在两个输入区下方，全宽显示
4. 对比算法基于 JSON 结构化比较（非纯文本逐行比较）：
   - 相同路径下的相同值标记为"相同"
   - JSON B 中新增的键标记为 ADD
   - JSON B 中缺少的键标记为 DEL
   - 相同键但不同值标记为 MOD
5. 数组比较：按索引逐一比较元素
6. 嵌套对象递归比较

### 3.5 统计栏与图例规则

1. 统计栏格式固定：`相同: N | 新增: N | 删除: N | 修改: N`
2. 各统计数字颜色与对应差异标记颜色一致
3. 图例始终可见，显示在差异结果区旁边
4. 图例内容：
   - 绿色方块 + "新增"（ADD）
   - 红色方块 + "删除"（DEL）
   - 黄色方块 + "修改"（MOD）
5. 统计栏仅在有对比结果时显示

### 3.6 JSONPath 查询规则

1. JSONPath 表达式语法遵循标准规范（如 `$.key`、`$[*]`、`$..key`）
2. 表达式输入为空时点击查询 → 提示"请输入 JSONPath 表达式"
3. 查询结果按匹配顺序排列
4. 每个匹配项显示完整路径和对应值
5. 匹配的值为对象或数组时，以格式化 JSON 展示

### 3.7 反序列化代码生成规则

1. 生成的代码应包含完整的类型定义（类名、字段名、字段类型）
2. 类型映射根据目标语言规范处理：
   - JSON string → 语言对应的字符串类型
   - JSON number → 根据是否有小数选择整数或浮点类型
   - JSON boolean → 语言对应的布尔类型
   - JSON null → 语言对应的可空类型或 Optional
   - JSON array → 语言对应的列表/数组类型
   - JSON object → 递归生成嵌套类型
3. 嵌套 JSON 对象生成独立的子类型定义
4. 字段命名遵循目标语言的惯用风格（如 Java 用 camelCase，Go 用 PascalCase）
5. 目标语言选择器**重复选择当前已选中的语言则忽略**，不重复触发生成

### 3.8 输出区交互规则

1. 格式化输出区使用语法高亮渲染（`<pre><code>` + CSS 类）；压缩输出区使用纯文本显示
2. 两个输出面板并排显示（左右布局），宽度各占 50%
2. 用户可选中输出区文本进行复制（不依赖复制按钮）
3. 复制时复制纯文本，不包含 HTML 标签和颜色信息
4. 输出区不支持编辑（只读）
5. JSON Diff 的差异结果区同样支持选中复制
