# IX-02 首页交互流程

> 对应页面：`02-home.html`
> 覆盖范围：首页欢迎区域、最近使用记录、分类工具卡片网格

---

## 一、界面状态定义

| 状态名称 | 触发条件 | 界面表现 |
|---------|---------|---------|
| 初始态 | 页面首次加载 | 显示欢迎标题、"最近使用"区域（从历史记录填充或显示空提示）、所有分类工具卡片网格 |

### 各区域界面元素

**欢迎区域**

| 元素 | 说明 |
|------|------|
| 标题 "开发者工具箱" | 页面顶部主标题 |
| 副标题 | 标题下方，描述应用用途的简要说明文字 |

**最近使用区域**

| 元素 | 说明 |
|------|------|
| 区域标题 "最近使用" | 左对齐标题 |
| 工具卡片网格 | 最多 4 张卡片，每张包含：工具图标 + 工具名称 + 简要描述 |
| 空状态提示 | 无历史记录时显示"暂无使用记录"提示文案 |

**分类区域**

各分类按顺序排列，每个分类包含：

| 分类名称 | 包含工具数 | 子工具列表 |
|---------|-----------|-----------|
| 字符编码 | 5 | Hex 转换、Base64 编解码、ASCII 转换、URL 编解码、Unicode 编解码 |
| JSON 工具 | 8 | Pretty 格式化、JSON 压缩、反序列化、JSONPath 查询、JSON Diff 等共 8 项 |
| 加密解密 | 5 | 包含常见加密/解密/哈希工具 |
| 数字计算 | 3 | 数值转换与计算工具 |
| 二维码 | 2 | 二维码生成与解析 |
| HTTP Client | 1 | HTTP 请求调试工具 |
| 时间计算 | 2 | 时间戳转换与日期计算 |
| Cron | 1 | Cron 表达式编辑与验证 |
| 正则调试 | 1 | 正则表达式实时测试 |
| Grok | 1 | Grok 模式调试工具 |
| Nginx | 1 | Nginx 配置生成/分析 |
| 配置转换 | 1 | 配置格式互转工具 |
| 编码解码 | 3 | 其他编码/解码工具 |

每张工具卡片包含：工具图标（SVG 或图标字体）+ 工具名称 + 一句话功能描述。

## 二、操作流程

### 2.1 页面加载流程

1. 渲染欢迎区域（标题 + 副标题）
2. 加载"最近使用"区域：
   - 从本地存储（localStorage 或 Tauri 持久化层）读取工具使用历史
   - 取最近使用的 4 个工具，生成卡片
   - 若无历史记录，显示空状态提示"暂无使用记录"
3. 按顺序渲染所有 13 个分类区域
4. 每个分类区域渲染标题 + 该分类下的工具卡片网格

### 2.2 工具卡片点击流程

1. 用户点击任意工具卡片（最近使用区或分类区）
2. 记录该工具的使用时间到本地存储（用于最近使用排序）
3. 页面导航到对应工具的 HTML 页面（如 `03-encoding.html`）
4. 导航方式为 Tauri 窗口内的页面跳转，非新窗口打开

### 2.3 最近使用更新流程

1. 每次用户通过首页或侧栏进入某个工具页面 → 记录一条使用记录
2. 使用记录包含：工具标识、工具名称、使用时间戳
3. 最近使用列表按时间倒序排列，最多保留 4 条
4. 同一工具多次使用时，仅保留最新一条记录（去重）
5. 首页每次加载时重新从存储读取并渲染

### 2.4 响应式布局适配流程

1. 页面宽度 >= 1200px → 卡片网格显示 4 列
2. 页面宽度 768px ~ 1199px → 卡片网格显示 3 列
3. 页面宽度 < 768px → 卡片网格显示 2 列
4. 最近使用区域最多 4 张卡片，跟随相同的列数规则
5. 布局变化时使用 CSS Grid 或 Flexbox 自动适配，无闪烁

## 三、交互规则

### 3.1 工具卡片规则

| 规则 | 说明 |
|------|------|
| 卡片悬停 | 鼠标悬停时卡片有轻微上浮阴影效果，光标变为 pointer |
| 卡片点击 | 点击整个卡片区域均可触发导航，不限于文字 |
| 卡片链接 | 每张卡片关联对应工具的 HTML 页面路径 |
| 卡片图标 | 使用统一的 SVG 图标或图标字体，尺寸一致 |

### 3.2 最近使用规则

1. 最近使用数据存储在客户端本地，不上传服务器
2. 存储键名固定，格式为 `{ toolId, toolName, timestamp }` 的数组
3. 最多保留 50 条历史记录（用于计算），仅展示最近 4 条
4. 应用卸载或清除浏览器数据后，最近使用记录清空
5. 最近使用区域卡片与分类区卡片视觉样式一致，仅位置不同

### 3.3 分类展示规则

1. 13 个分类按固定顺序排列，不根据使用频率调整顺序
2. 每个分类的标题与侧栏分类名称保持一致
3. 每个分类内的工具按预定义顺序排列
4. 各分类的工具卡片网格独立，不跨分类合并
5. 分类之间有明确的视觉分隔（间距或分隔线）

### 3.4 响应式规则

1. 列数变化通过 CSS 媒体查询控制，优先使用 `auto-fill` / `auto-fit` 配合 `minmax`
2. 卡片最小宽度不低于 200px，避免内容挤压
3. 欢迎标题在窄屏下字号适当缩小
4. 滚动行为：页面整体垂直滚动，各区域不独立滚动

### 3.5 导航与页面跳转规则

1. 首页作为应用默认着陆页，可通过面包屑"首页"或点击 Logo 回到首页
2. 所有工具卡片点击后均在当前窗口内导航，不打开新窗口
3. 导航使用 Tauri 提供的页面路由机制，保持窗口状态连续
4. 页面跳转时不需要确认（无未保存数据风险）

### 3.6 空状态与容错规则

1. 最近使用区域无数据时显示"暂无使用记录"文案，不隐藏整个区域
2. 工具卡片图标加载失败时显示占位图标，不影响文字展示
3. 本地存储不可用时，最近使用区域降级为空状态，不影响其他功能
4. 工具数量或分类配置变更后，首页应自动适应，不硬编码布局
