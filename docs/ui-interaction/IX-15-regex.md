# IX-15 正则调试交互流程

> 页面文件：`15-regex.html`

## 界面状态定义

| 状态名称 | 触发条件 | 界面表现 |
|---------|---------|---------|
| 初始态 | 页面首次加载，正则输入框和测试字符串均为空 | 正则输入行显示 `/ /` 占位符，flag 按钮全部未选中（灰色），测试字符串文本域显示 placeholder "请输入测试字符串…"，匹配结果区域显示空状态提示 "输入正则表达式和测试字符串开始调试"，匹配数量显示为 `0`，常用正则库折叠收起 |
| 匹配态 | 正则表达式合法且在测试字符串中找到匹配项 | 测试字符串中匹配部分以黄色高亮背景标注，匹配结果区域显示匹配总数（如 "找到 3 个匹配"），匹配组选项卡可切换查看每个匹配的捕获组详情，每个捕获组显示组号、名称（如有命名组）和匹配内容 |
| 无匹配态 | 正则表达式合法但未在测试字符串中找到匹配项 | 测试字符串无高亮，匹配结果区域显示 "未找到匹配" 提示（灰色文字），匹配数量显示为 `0`，匹配组选项卡置灰不可点击 |
| 异常态 | 正则表达式语法错误（未闭合括号、非法转义等） | 正则输入框边框变为红色，输入框下方显示红色错误提示信息（如 "Unmatched '(' at position 5"），匹配结果区域清空，测试字符串不高亮，flag 切换仍可用 |

## 操作流程

1. 用户在正则输入行中间的文本框输入正则表达式，左右两侧自动显示 `/` 分隔符。
2. 用户点击 flag 按钮（`g` 全局 / `i` 忽略大小写 / `m` 多行 / `s` 单行点号）切换开启或关闭，已选中的 flag 按钮显示为高亮（蓝色），未选中为灰色。
3. 用户在测试字符串文本域输入或粘贴待匹配的文本内容（支持多行）。
4. 系统实时监听正则表达式和测试字符串的变化，每次输入变动后自动执行匹配，无需手动触发。
5. 若正则合法且存在匹配，进入 **匹配态**：测试字符串中所有匹配项以黄色背景高亮，匹配结果区域显示匹配总数。
6. 若正则合法但无匹配，进入 **无匹配态**：显示 "未找到匹配" 提示。
7. 若正则语法错误，进入 **异常态**：输入框下方显示错误信息，不执行匹配。
8. 用户点击 **匹配组** 选项卡，展开每个匹配项的捕获组列表，显示组号、命名（如有）和匹配内容。
9. 用户展开 **常用正则** 库区域，显示预设正则列表（邮箱、URL、IPv4、手机号、身份证号、日期等），每项显示名称和简短说明。
10. 用户点击常用正则列表中的某一项，系统自动将该正则填入正则输入框，并设置推荐的 flag 组合，随后自动触发匹配。
11. 用户修改正则或测试字符串后，系统重新匹配并更新高亮和结果区域。

## 交互规则

| 规则编号 | 规则描述 | 去重保护 |
|---------|---------|---------|
| R15-01 | 正则匹配采用实时触发机制：正则输入框或测试字符串文本域内容变化后 300ms 防抖延迟执行匹配，避免高频输入时性能问题。 | — |
| R15-02 | Flag 按钮为独立切换开关，可任意组合；点击已选中的 flag 取消选中，点击未选中的 flag 开启选中，切换后立即重新匹配。 | — |
| R15-03 | 正则输入框失去焦点时进行语法校验；若语法错误，输入框边框变红并显示错误信息，不清空已有输入内容。 | — |
| R15-04 | 匹配高亮采用 `<mark>` 标签渲染，确保不破坏原始文本的换行和空白结构；多个匹配项相邻时不重叠渲染。 | — |
| R15-05 | 常用正则库中的预设项点击后自动覆盖当前正则输入框内容和 flag 设置，若测试字符串已有内容则立即显示匹配结果。 | —（便捷按钮） |
| R15-06 | 匹配组选项卡在无匹配态时禁用，不响应点击；匹配态下默认显示第一个匹配项的捕获组，用户可切换查看其他匹配项；**重复点击当前已展开的匹配组选项卡则忽略**。 | — |
| R15-07 | 正则输入框支持键盘快捷键：`Ctrl+Enter` 强制执行匹配（绕过防抖）；测试字符串文本域支持 `Tab` 键输入制表符而非切换焦点。 | — |
| R15-08 | 页面状态（正则表达式、flag、测试字符串）在会话内保持，刷新页面后恢复为初始态。 | — |
| R15-09 | 常用正则库默认折叠收起，点击标题栏展开/收起；展开时显示全部预设项列表，收起时仅显示标题栏。 | — |
| R15-10 | 匹配结果区域在匹配态下显示匹配总数和第一个匹配的起止位置；测试字符串文本域中匹配项高亮颜色与匹配组选项卡中的编号对应。 | — |
