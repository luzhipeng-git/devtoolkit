# IX-06 RSA 工具交互流程

> 涵盖页面：`06-crypto-rsa.html`

---

## 一、界面状态定义

| 状态名称 | 说明 | 界面表现 |
|---------|------|---------|
| 初始态 | 页面加载完成 | 「密钥生成」Tab 默认选中；密钥长度默认 2048；公钥指数默认 65537；密钥格式默认 PKCS#8；公钥输出区为空；私钥输出区为空并默认遮罩 |
| 密钥生成态 | 当前 Tab 为密钥生成 | 显示密钥参数选项（密钥长度、公钥指数、密钥格式）；「生成密钥对」按钮可用；公钥/私钥输出区可见 |
| 密钥生成中态 | 正在生成密钥对 | 「生成密钥对」按钮显示 loading spinner 并禁用；输出区显示「正在生成密钥对...」；所有参数选项禁用 |
| 密钥生成完成态 | 密钥对生成成功 | 公钥输出区显示 PEM 格式公钥（readonly）；私钥输出区显示遮罩后的私钥；可切换私钥可见性 |
| 加密态 | 加密/解密 Tab → 加密子 Tab | 显示明文输入区 + 公钥（PEM）输入区 + 「加密」按钮 + Base64 结果输出区 |
| 解密态 | 加密/解密 Tab → 解密子 Tab | 显示密文（Base64）输入区 + 私钥（PEM）输入区 + 「解密」按钮 + 明文结果输出区 |
| 签名态 | 签名/验签 Tab → 签名子 Tab | 显示数据输入区 + 私钥输入区 + 算法选择（SHA-256/384/512）+ 「签名」按钮 + 签名结果（Base64）输出区 |
| 验签态 | 签名/验签 Tab → 验签子 Tab | 显示原始数据输入区 + 签名（Base64）输入区 + 公钥输入区 + 「验签」按钮 + 验证结果区域 |
| 执行中态 | 用户点击任一操作按钮 | 对应按钮显示 loading spinner 并禁用；其余输入区域不变；输出区显示「处理中...」 |
| 结果态 | 操作执行成功 | 加密/解密/签名：输出区显示结果文本，带复制按钮；验签：显示绿色「验证通过」或红色「验证失败」结果框 |
| 异常态 | 操作执行失败 | 在输出区域上方显示红色错误横幅，包含具体错误原因；操作按钮恢复可用 |
| 私钥隐藏态 | 私钥显示默认状态 | 私钥 textarea 内容以圆点遮罩；眼睛图标为闭眼状态 |
| 私钥可见态 | 用户点击私钥眼睛图标 | 私钥 textarea 内容以 PEM 明文显示；眼睛图标为睁眼状态 |

## 二、操作流程

### 2.1 密钥生成流程

1. 用户在「密钥生成」Tab 下选择参数：密钥长度（2048/4096）、公钥指数（65537/3/17）、密钥格式（PKCS#1/PKCS#8）
2. 用户点击「生成密钥对」按钮
3. 界面进入密钥生成中态，按钮显示 loading spinner
4. 系统使用 Web Crypto API 生成 RSA 密钥对（RSA-OAEP 用于加密/解密，RSASSA-PKCS1-v1_5 用于签名/验签）
5. 生成完成后，公钥输出区显示 PEM 格式公钥文本（readonly textarea，可滚动查看）
6. 私钥输出区默认以遮罩方式显示，用户可点击眼睛图标切换可见性
7. 输出区下方显示安全提示框：「私钥是敏感信息，请妥善保管，切勿泄露」
8. 用户可点击公钥/私钥输出区右上角的「复制」按钮，将 PEM 内容复制到剪贴板

### 2.2 RSA 加密流程

1. 用户切换到「加密/解密」Tab，选择「加密」子 Tab
2. 在明文输入区输入待加密的文本内容
3. 在公钥（PEM）输入区提供接收方的 RSA 公钥，支持以下输入方式：
   - **手动粘贴**：在 textarea 中直接粘贴 PEM 格式公钥
   - **文件上传**：点击「上传文件」按钮，选择 .pem / .cer / .crt / .pub 文件，系统自动读取内容并填入
   - **从本页生成结果复制**：若当前会话中已生成密钥对，可一键填入公钥
4. 用户点击「加密」按钮
5. 界面进入执行中态
6. 系统使用 RSA-OAEP 算法加密明文
7. 加密成功后，输出区显示 Base64 编码的密文
8. 用户可点击「复制」按钮复制密文

### 2.3 RSA 解密流程

1. 用户切换到「加密/解密」Tab，选择「解密」子 Tab
2. 在密文输入区粘贴 Base64 编码的密文
3. 在私钥（PEM）输入区提供对应的 RSA 私钥，支持以下输入方式：
   - **手动粘贴**：在 textarea 中直接粘贴 PEM 格式私钥
   - **文件上传**：点击「上传文件」按钮，选择 .pem / .key 文件，系统自动读取内容并填入
   - **从本页生成结果复制**：若当前会话中已生成密钥对，可一键填入私钥
4. 用户点击「解密」按钮
5. 界面进入执行中态
6. 系统使用 RSA-OAEP 算法解密密文
7. 解密成功后，输出区显示明文
8. 解密失败（私钥不匹配、密文格式错误等）进入异常态

### 2.4 RSA 签名流程

1. 用户切换到「签名/验签」Tab，选择「签名」子 Tab
2. 在数据输入区输入待签名的原始数据
3. 在私钥输入区提供签名方的 RSA 私钥（支持手动粘贴或文件上传）
4. 在算法下拉框中选择哈希算法（SHA-256 / SHA-384 / SHA-512，默认 SHA-256）
5. 用户点击「签名」按钮
6. 界面进入执行中态
7. 系统使用 RSASSA-PKCS1-v1_5 算法和所选哈希算法生成签名
8. 签名成功后，输出区显示 Base64 编码的签名值
9. 用户可点击「复制」按钮复制签名

### 2.5 RSA 验签流程

1. 用户切换到「签名/验签」Tab，选择「验签」子 Tab
2. 在原始数据输入区输入原始数据
3. 在签名输入区粘贴 Base64 编码的签名值
4. 在公钥输入区提供签名方的 RSA 公钥（支持手动粘贴或文件上传）
5. 确认算法与签名时使用的算法一致
6. 用户点击「验签」按钮
7. 界面进入执行中态
8. 系统使用公钥验证签名
9. 验证成功：显示绿色结果框，文字「验证通过」，带勾选图标
10. 验证失败：显示红色结果框，文字「验证失败」，带叉号图标，并注明可能原因（数据被篡改、密钥不匹配、签名格式错误等）

## 三、交互规则

| 规则编号 | 触发条件 | 行为描述 | 去重保护 | 约束说明 |
|---------|---------|---------|---------|---------|
| R-06-01 | 主 Tab 切换（密钥生成/加密解密/签名验签） | 切换到对应 Tab 内容区；保留各 Tab 内已有的输入数据；子 Tab 默认选中第一个（加密/签名） | **重复点击当前已激活的 Tab 则忽略**，不做任何状态变更或内容切换 | Tab 切换不丢失任何已输入数据 |
| R-06-02 | 子 Tab 切换（加密↔解密 / 签名↔验签） | 切换到对应子 Tab 内容区；清空结果输出区；保留输入区内容 | **重复点击当前已激活的子 Tab 则忽略**，不重复清空输出 | 仅清空输出，保留输入 |
| R-06-03 | 点击「生成密钥对」 | 校验无特殊要求；进入密钥生成中态；调用 Web Crypto API 生成密钥对；完成后更新输出区 | 无需去重（操作按钮有明确意图） | 4096 位密钥生成可能需要数秒，期间显示 loading 状态 |
| R-06-04 | 密钥长度变更 | 若已有生成结果，弹出确认对话框「更改密钥长度将清除当前密钥对，是否继续？」；确认后清空公钥/私钥输出区 | **重复选择当前密钥长度则忽略**，不弹出确认对话框 | 防止用户误操作丢失已生成的密钥 |
| R-06-05 | 公钥指数变更 | 无即时效果，下次生成时使用新的公钥指数 | **重复选择当前公钥指数则忽略** | 不影响已有结果 |
| R-06-06 | 密钥格式变更 | 若已有生成结果，弹出确认对话框「更改密钥格式将清除当前密钥对，是否继续？」；确认后清空输出区并重新以新格式生成 | **重复选择当前密钥格式则忽略**，不弹出确认对话框 | PKCS#1 与 PKCS#8 格式不兼容 |
| R-06-07 | 私钥眼睛图标点击 | 切换私钥 textarea 的显示/隐藏状态 | 无需去重（toggle 开关，每次翻转状态） | 仅影响显示，不影响实际值 |
| R-06-08 | 点击加密按钮 | 校验：明文非空、公钥非空且格式合法（PEM 头尾标记）；校验通过执行加密；校验失败在对应输入框下方显示红色错误提示 | 无需去重（操作按钮有明确意图，且执行中会禁用） | RSA 加密有长度限制（密钥长度 - 填充开销），超长时提示分块或使用混合加密 |
| R-06-09 | 点击解密按钮 | 校验：密文非空且为合法 Base64、私钥非空且格式合法；校验通过执行解密；校验失败显示错误信息 | 无需去重（操作按钮有明确意图，且执行中会禁用） | 解密失败常见原因：私钥不匹配、密文被篡改、Base64 格式错误 |
| R-06-10 | 点击签名按钮 | 校验：数据非空、私钥非空且格式合法、算法已选择；校验通过执行签名 | 无需去重（操作按钮有明确意图，且执行中会禁用） | 算法默认 SHA-256 |
| R-06-11 | 点击验签按钮 | 校验：原始数据非空、签名非空且为合法 Base64、公钥非空且格式合法；校验通过执行验证 | 无需去重（操作按钮有明确意图，且执行中会禁用） | 验签结果以醒目颜色框显示，不可复制 |
| R-06-12 | 验签结果展示 | 通过：绿色背景框 + 白色勾选图标 + 「验证通过」文字；失败：红色背景框 + 白色叉号图标 + 「验证失败」文字 + 可能原因说明 | —（展示规则，不涉及用户主动选择） | 结果框始终显示，不自动消失 |
| R-06-13 | 点击复制按钮 | 将对应输出区内容复制到剪贴板；按钮图标临时变为勾选 + 「已复制」文字；2 秒后恢复 | —（操作按钮，不涉及选择型控件） | 公钥区、私钥区、加密结果区、签名结果区各有独立复制按钮 |
| R-06-14 | 安全提示框 | 在密钥生成 Tab 的私钥输出区下方始终显示黄色提示框：「私钥是敏感信息，请妥善保管，切勿泄露」 | —（展示规则，不涉及用户主动选择） | 提示框不可关闭，始终可见 |
| R-06-15 | PEM 格式校验 | 用户输入公钥/私钥时，检查 PEM 头尾标记（-----BEGIN PUBLIC KEY----- / -----BEGIN PRIVATE KEY----- 等）；格式不合法时输入框边框变红并显示提示 | —（校验规则，不涉及选择型控件） | 实时校验，在输入时即反馈 |
| R-06-16 | RSA 加密长度限制 | 当明文长度超过 RSA-OAEP 最大加密长度（密钥字节数 - 42）时，加密按钮点击后在明文输入框下方显示警告：「数据过长，RSA 加密最大支持 X 字节。建议使用 AES 混合加密」 | —（校验规则，不涉及选择型控件） | 2048 位密钥最多加密 214 字节，4096 位最多 470 字节 |
| R-06-17 | 密钥/证书文件上传 | 公钥/私钥输入区均支持文件上传，点击「上传文件」按钮后打开系统文件选择对话框；支持的文件格式：.pem、.key、.cer、.crt、.pub；文件读取后自动填入对应的 textarea，用户可编辑；文件大小限制 1MB | —（操作按钮，不涉及选择型控件） | 文件内容会进行 PEM 格式校验，非 PEM 格式文件提示「文件内容不是有效的 PEM 格式」 |
