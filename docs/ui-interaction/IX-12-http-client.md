# IX-12 HTTP Client 交互流程

> 页面文件：`12-http-client.html`
> 功能模块：HTTP 请求调试工具

---

## 界面状态定义

| 状态名称 | 触发条件 | 界面表现 |
|---------|---------|---------|
| 初始态 | 页面首次加载 | 请求栏：方法选择器默认选中 GET（绿色高亮）；URL 输入框为空并显示占位提示"输入请求 URL"；"发送"按钮可用但置灰样式（URL 为空时不可点击）；请求配置标签页默认显示 Params；Body 标签页子选项默认选中"none"；响应区域显示空状态占位提示"发送请求后查看响应结果" |
| 配置态 | 用户输入 URL 并编辑请求参数 | 请求栏：URL 输入框显示用户输入的内容；方法选择器高亮当前选中方法；"发送"按钮可点击（URL 非空时）；请求配置标签页可切换和编辑；响应区域保持上次结果或空状态占位 |
| 发送中 | 用户点击"发送"按钮后请求进行中 | 请求栏："发送"按钮显示加载旋转图标并变为不可点击状态；按钮文字临时变为"发送中..."；响应区域显示加载动画或进度指示；请求配置区域不可编辑（可选锁定） |
| 响应态 | 请求完成并收到响应 | 响应区域显示：状态码徽章（2xx 绿色、3xx 蓝色、4xx 橙色、5xx 红色）；响应时间（如"234ms"）；响应体大小（如"1.2 KB"）；响应标签页默认显示 Body；Body 标签页显示格式化后的响应内容（JSON 自动语法高亮，其他类型显示原始文本）；Headers 和 Cookies 标签页可切换查看；请求栏恢复正常可交互状态 |
| 异常态 | 请求失败（网络错误、DNS 解析失败、连接超时、SSL 错误等） | 响应区域显示错误提示卡片：包含错误类型图标、错误描述信息（如"连接超时：请求在 30 秒内未收到响应"）、重试按钮；请求栏恢复正常可交互状态 |
| SSL 单向认证态 | Auth 标签页选择「SSL/TLS 证书」→「单向认证」 | 显示 CA 证书配置区：支持文本粘贴（textarea）或文件上传（.pem / .crt / .cer）；下方显示「跳过 SSL 证书验证」开关（默认关闭，启用后以黄色警告提示安全风险）；URL 以 https:// 开头时发送请求将使用配置的 CA 证书验证服务端证书 |
| SSL 双向认证态 | Auth 标签页选择「SSL/TLS 证书」→「双向认证」 | 显示三组证书配置区：（1）客户端证书 — 支持文本粘贴或文件上传（.pem / .crt / .cer）；（2）客户端私钥 — 支持文本粘贴或文件上传（.pem / .key），默认遮罩显示，可切换可见性；（3）CA 证书（可选）— 支持文本粘贴或文件上传；下方显示「跳过 SSL 证书验证」开关（默认关闭） |

## 操作流程

1. 用户在请求栏选择 HTTP 方法，方法选择器以不同颜色徽章标识（GET 绿色/POST 蓝色/PUT 橙色/DELETE 红色/PATCH 紫色），选中项高亮显示。
2. 用户在 URL 输入框中输入请求地址，支持包含查询参数的完整 URL。
3. 用户可切换请求配置标签页（Params、Headers、Body、Auth）编辑请求参数。
4. 切换到 Params 标签页时，系统自动从 URL 中提取已有查询参数并填充为键值对列表，用户可新增、编辑、删除参数。
5. 切换到 Headers 标签页时，显示请求头键值对编辑器，预填充常用头（如 Content-Type），用户可新增、编辑、启用/禁用、删除请求头。
6. 切换到 Body 标签页时，显示子选项（none/JSON/Form/Raw/Binary）：选择"none"不发送请求体；选择"JSON"显示 JSON 键值对编辑器并自动设置 Content-Type；选择"Form"显示表单键值对编辑器；选择"Raw"显示纯文本编辑区域；选择"Binary"显示文件上传入口。
7. 切换到 Auth 标签页时，显示认证配置选项（None / Basic Auth / Bearer Token / SSL/TLS 证书）：
   - **None**：不使用任何认证机制。
   - **Basic Auth**：显示用户名和密码输入框，请求时自动在 Header 中添加 `Authorization: Basic <encoded>`。
   - **Bearer Token**：显示 Token 输入框，请求时自动添加 `Authorization: Bearer <token>`。
   - **SSL/TLS 证书**：显示子选项（单向认证 / 双向认证）：
     - **单向认证（验证服务端）**：用户配置 CA 证书（用于验证服务端证书的合法性），支持文本粘贴或文件上传。适用于自签名证书或企业内部 CA 环境。下方提供「跳过 SSL 证书验证」开关，启用后不验证服务端证书（以黄色警告标注安全风险）。
     - **双向认证（mTLS）**：用户需配置三份证书材料：
       1. 客户端证书（.pem / .crt / .cer）— 支持文本粘贴或文件上传
       2. 客户端私钥（.pem / .key）— 支持文本粘贴或文件上传，默认遮罩显示
       3. CA 证书（可选，.pem / .crt / .cer）— 用于验证服务端证书，支持文本粘贴或文件上传
     下方同样提供「跳过 SSL 证书验证」开关。
8. 用户点击"发送"按钮，系统进入发送中状态，向后端发送 HTTP 请求。
9. 请求完成后，系统进入响应态，在响应区域显示状态码、响应时间、响应体大小，并在 Body 标签页中展示格式化的响应内容。
10. 用户可切换响应标签页（Body、Headers、Cookies）查看不同部分的信息。
11. 用户可通过 cURL 导入功能，在导入区域粘贴 cURL 命令，系统解析后自动填充方法、URL、Headers、Body 等字段。
12. 用户可通过 cURL 导出功能，将当前配置的请求导出为等效的 cURL 命令，并支持一键复制。

## 交互规则

| 交互动作 | 触发元素 | 响应行为 | 去重保护 | 约束条件 |
|---------|---------|---------|---------|---------|
| 选择方法 | 方法选择器徽章 | 切换高亮至所选方法，更新徽章颜色；若方法为 GET/DELETE 则 Body 标签页提示"GET/DELETE 请求通常不包含请求体"但仍允许编辑 | **重复选择当前 HTTP 方法则忽略** | 方法切换不清除已编辑的其他配置 |
| 输入 URL | URL 输入框（`input` 事件） | 实时校验 URL 格式；URL 非空时"发送"按钮变为可点击状态；自动提取查询参数同步到 Params 标签页 | — | URL 必须以 http:// 或 https:// 开头（可自动补全） |
| 切换请求标签页 | Params/Headers/Body/Auth 标签 | 切换到对应配置面板，保留各标签页的编辑状态 | **重复点击当前已激活的标签页则忽略** | 标签页切换不丢失已编辑数据 |
| 编辑键值对 | Params/Headers/Body 键值对编辑器 | 支持新增行（点击"+"按钮或回车）、删除行（点击删除图标）、禁用单行（切换开关） | — | 键名不可为空；重复键名显示警告 |
| 切换 Body 类型 | Body 子选项（none/JSON/Form/Raw/Binary） | 切换显示对应的编辑器界面；JSON/Form 模式自动设置 Content-Type 请求头 | **重复选择当前 Body 类型则忽略**，不弹出确认清空对话框 | 切换 Body 类型时若已有内容，提示用户确认是否清空 |
| 发送请求 | "发送"按钮 | 校验 URL 非空后发送请求；按钮进入加载状态；响应到达后更新响应区域 | — | URL 为空时按钮不可点击；发送中不可重复点击 |
| 切换响应标签页 | Body/Headers/Cookies 标签 | 切换响应区域显示内容；Body 显示格式化响应体；Headers 显示响应头列表；Cookies 显示 Cookie 列表 | **重复点击当前已激活的响应标签页则忽略** | 仅在响应态时可切换 |
| JSON 语法高亮 | Body 响应内容 | 自动检测 JSON 响应并进行语法高亮（键名蓝色、字符串绿色、数字橙色、布尔值紫色、null 灰色） | — | 非 JSON 响应显示原始文本 |
| cURL 导入 | cURL 导入入口 | 解析 cURL 命令字符串，提取方法、URL、Headers、Body 并填充到对应字段 | — | 支持标准 cURL 语法；解析失败时显示具体错误位置 |
| cURL 导出 | cURL 导出按钮 | 根据当前请求配置生成等效 cURL 命令，显示在弹出框中并提供复制按钮 | — | 包含所有已配置的方法、URL、Headers、Body、Auth 信息 |
| 复制响应内容 | 响应区域复制按钮 | 将当前响应标签页的内容写入系统剪贴板并显示反馈 | — | 反馈持续 2 秒后恢复 |
| 超时处理 | 请求超过配置的超时时间 | 中止请求并进入异常态，显示超时错误信息和重试按钮 | — | 默认超时 30 秒，可在设置中调整 |
| Auth 类型切换 | Auth 标签页认证类型选择器（None/Basic/Bearer/SSL证书） | 切换显示对应的认证配置界面；切换类型不清除已填写的其他类型配置，切回时可恢复 | **重复选择当前认证类型则忽略** | 不同认证类型的配置独立存储 |
| SSL 单/双向认证子选项切换 | SSL/TLS 证书下的「单向认证/双向认证」子选项 | 单向认证：显示 CA 证书配置区；双向认证：显示客户端证书、客户端私钥、CA 证书三组配置区；切换不清除已填写的证书内容 | **重复选择当前认证模式则忽略** | 单向认证仅需 CA 证书；双向认证需客户端证书和私钥（必填）+ CA 证书（可选） |
| SSL 证书文件上传 | 证书/私钥配置区的「上传文件」按钮 | 打开系统文件选择对话框，选择证书或私钥文件后自动读取内容填入对应 textarea；支持的格式：.pem、.crt、.cer、.key、.p12、.pfx | — | 文件大小限制 1MB；.p12/.pfx 格式需额外输入密码解密后提取证书和私钥 |
| 跳过 SSL 证书验证开关 | 「跳过 SSL 证书验证」开关 | 关闭（默认）：发送 https 请求时验证服务端证书；开启：跳过证书验证，以黄色警告文字提示「跳过证书验证存在安全风险，仅建议在测试环境使用」 | 无需去重（toggle 开关，每次翻转状态） | 开启时请求可能存在中间人攻击风险 |
| SSL 证书校验失败 | https 请求证书验证不通过 | 进入异常态，响应区域显示 SSL 错误信息（如「证书已过期」「证书域名不匹配」「无法验证证书链」「客户端证书无效」等），提供重试按钮 | — | 错误信息应包含具体失败原因，便于排查 |
