# IX-16 Grok 调试交互流程

> 页面文件：`16-grok.html`

## 界面状态定义

| 状态名称 | 触发条件 | 界面表现 |
|---------|---------|---------|
| 初始态 | 页面首次加载，pattern 输入框和样本数据文本域均为空 | Pattern 输入框显示 placeholder `%{PATTERN:name}`，"解析" 按钮为禁用态（灰色），样本数据文本域显示 placeholder "请输入日志数据…"，结果区域显示空状态提示 "输入 Grok 模式和日志数据后点击解析"，常用模式库折叠收起，自定义模式区域折叠收起 |
| 匹配态 | Grok 模式合法且完整匹配日志数据 | 结果区域以键值对表格显示提取结果，每行显示字段名称（左列）和对应值（右列），表格上方显示 "成功提取 N 个字段"，字段名以等宽字体显示，值以普通字体显示 |
| 部分匹配态 | Grok 模式合法但仅匹配日志数据的部分内容 | 结果区域显示已提取的键值对表格（字段名和值），表格下方以黄色提示栏显示 "部分匹配：已匹配至第 X 个字符，剩余内容未匹配"，未匹配部分在样本数据中以灰色标注 |
| 无匹配态 | Grok 模式合法但无法匹配日志数据 | 结果区域显示 "未找到匹配" 提示（灰色文字），建议用户检查模式与数据格式是否一致 |
| 异常态 | Grok 模式语法错误（未定义的模式名称、未闭合的 `%{}` 等） | Pattern 输入框边框变红，输入框下方显示红色错误提示（如 "未知的 pattern: FOOBAR" 或 "未闭合的 `%{`"），结果区域清空 |

## 操作流程

1. 用户在 Pattern 输入框输入 Grok 模式（如 `%{IP:client} %{WORD:method} %{URIPATH:path}`）。
2. 用户在样本数据文本域输入或粘贴日志行（支持多行，每行独立解析）。
3. "解析" 按钮在 Pattern 和样本数据均非空时自动变为可点击状态（蓝色）。
4. 用户点击 "解析" 按钮，系统对样本数据的每一行执行 Grok 解析。
5. 若模式合法且完整匹配，进入 **匹配态**：结果区域以表格形式展示提取的键值对。
6. 若模式合法但仅匹配部分内容，进入 **部分匹配态**：显示已提取字段并提示未匹配的剩余内容。
7. 若模式合法但完全无法匹配，进入 **无匹配态**：显示未匹配提示。
8. 若模式语法错误，进入 **异常态**：在输入框下方显示错误信息。
9. 用户展开 **常用模式库**，查看内置 Grok 模式列表（IP、HOSTNAME、WORD、NUMBER、TIMESTAMP_ISO8601、SYSLOGBASE、LOGLEVEL、URI、QUOTEDSTRING 等），每项显示模式名称和简短说明。
10. 用户点击常用模式库中的某一项，系统将 `%{PATTERN_NAME}` 插入到 Pattern 输入框光标当前位置。
11. 用户展开 **自定义模式** 区域，可输入自定义模式定义（如 `MYAPP [A-Z]+-\d+`），点击 "保存" 按钮将其添加到自定义模式列表，后续解析时可使用 `%{MYAPP:field}` 引用。
12. 用户可在自定义模式列表中查看已保存的模式，支持删除不再需要的自定义模式。

## 交互规则

| 规则编号 | 规则描述 | 去重保护 |
|---------|---------|---------|
| R16-01 | "解析" 按钮仅在 Pattern 输入框和样本数据文本域均有内容时可点击；任一为空时按钮显示为禁用态。 | —（Grok 工具以手动操作为主，模式库为插入操作非替换，不涉及破坏性选择） |
| R16-02 | 解析操作由用户手动点击 "解析" 按钮触发，不采用实时匹配，避免 Grok 解析的性能开销。 | —（Grok 工具以手动操作为主，模式库为插入操作非替换，不涉及破坏性选择） |
| R16-03 | 多行样本数据逐行独立解析，结果区域按行分组显示，每行前标注行号；解析失败的行单独标注红色错误提示。 | —（Grok 工具以手动操作为主，模式库为插入操作非替换，不涉及破坏性选择） |
| R16-04 | 常用模式库中的模式点击后以插入方式写入 Pattern 输入框光标位置，不覆盖已有内容；若输入框为空则直接填入。 | —（Grok 工具以手动操作为主，模式库为插入操作非替换，不涉及破坏性选择） |
| R16-05 | 自定义模式在当前会话内有效，保存在内存中；页面刷新后自定义模式丢失，需重新定义。 | —（Grok 工具以手动操作为主，模式库为插入操作非替换，不涉及破坏性选择） |
| R16-06 | 自定义模式名称必须以字母开头，仅包含字母、数字和下划线，长度不超过 50 个字符；不符合规则的名称在保存时提示错误。 | —（Grok 工具以手动操作为主，模式库为插入操作非替换，不涉及破坏性选择） |
| R16-07 | 部分匹配态下，样本数据文本域中已匹配的部分以绿色底色标注，未匹配的剩余部分以灰色底色标注，便于用户定位匹配边界。 | —（Grok 工具以手动操作为主，模式库为插入操作非替换，不涉及破坏性选择） |
| R16-08 | 结果表格中的字段值支持点击复制（单击选中，出现 "复制" 浮层按钮），方便用户提取特定值。 | —（Grok 工具以手动操作为主，模式库为插入操作非替换，不涉及破坏性选择） |
| R16-09 | Pattern 输入框支持语法高亮：`%{PATTERN:name}` 中 PATTERN 部分以蓝色显示，name 部分以绿色显示，普通文本以默认颜色显示，未闭合的 `%{` 以红色显示。 | —（Grok 工具以手动操作为主，模式库为插入操作非替换，不涉及破坏性选择） |
| R16-10 | 内置 Grok 模式库包含至少 30 个常用模式，涵盖网络（IP、MAC、CIDR）、时间（TIMESTAMP、DATE、TIME）、路径（URI、UNIXPATH）、日志（SYSLOGBASE、LOGLEVEL）等类别，按类别分组显示。 | —（Grok 工具以手动操作为主，模式库为插入操作非替换，不涉及破坏性选择） |
