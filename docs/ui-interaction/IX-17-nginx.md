# IX-17 Nginx 工具交互流程

> 页面文件：`17-nginx.html`

## 界面状态定义

| 状态名称 | 触发条件 | 界面表现 |
|---------|---------|---------|
| 初始态 | 页面首次加载，默认显示 "格式化" 选项卡 | 四个选项卡（格式化、语法检查、Diff 对比、模板）均可见，当前选中的 "格式化" 选项卡以蓝色下划线标识；左侧输入面板为空，显示 placeholder "请输入 Nginx 配置…"；右侧输出面板为空，显示空状态提示 "格式化后的配置将显示在这里"；中间 "格式化" 按钮为禁用态 |
| 格式化态 | 用户输入配置并点击 "格式化" 按钮 | 左侧面板保持原始输入内容，右侧面板显示格式化后的 Nginx 配置（统一缩进、对齐指令、规范空行），格式化结果带有语法高亮（指令为蓝色、参数为黑色、注释为灰色、字符串为绿色） |
| 检查通过态 | 用户在 "语法检查" 选项卡输入配置并点击 "检查" 按钮，且配置语法正确 | 输入面板中配置内容保持不变，面板下方显示绿色提示 "语法检查通过"，不标注任何错误行 |
| 检查失败态 | 用户在 "语法检查" 选项卡输入配置并点击 "检查" 按钮，且配置存在语法错误 | 输入面板中错误行以红色底色标注，行号列以红色高亮，面板下方显示错误列表（每个错误包含行号和错误描述，如 "第 12 行：缺少分号"） |
| Diff 态 | 用户在 "Diff 对比" 选项卡输入新旧配置并点击 "对比" 按钮 | 两个面板之间以行对行方式展示差异：新增行以绿色背景标注，删除行以红色背景标注，修改行以黄色背景标注，未变化行以默认背景显示；差异摘要显示在顶部（如 "新增 5 行，删除 2 行，修改 3 行"） |
| 模板态 | 用户切换到 "模板" 选项卡 | 显示预设模板卡片列表（反向代理、负载均衡、SSL/HTTPS、静态文件、缓存、重定向、WebSocket、安全头部等），每个卡片显示模板名称和简短描述；点击卡片后配置内容填入输入区域并自动切换到 "格式化" 选项卡 |

## 操作流程

1. 页面加载后默认显示 "格式化" 选项卡，用户可在四个选项卡之间自由切换。
2. **格式化流程**：用户在左侧面板输入或粘贴原始 Nginx 配置文本，点击中间的 "格式化" 按钮，系统在右侧面板显示格式化后的配置，带有语法高亮。
3. **语法检查流程**：用户切换到 "语法检查" 选项卡，在输入区域粘贴或输入 Nginx 配置，点击 "检查" 按钮。若语法正确，显示绿色通过提示；若存在错误，高亮错误行并列出错误详情。
4. **Diff 对比流程**：用户切换到 "Diff 对比" 选项卡，在左侧面板输入旧配置，右侧面板输入新配置，点击 "对比" 按钮。系统以行级差异对比展示结果，标注增删改。
5. **模板使用流程**：用户切换到 "模板" 选项卡，浏览预设模板列表，点击所需模板卡片，系统将该模板配置填入输入区域并自动切换回 "格式化" 选项卡，用户可在格式化结果基础上继续编辑。
6. 用户在任何选项卡的输入面板中均可自由编辑内容，修改后需再次点击对应的操作按钮重新执行。

## 交互规则

| 规则编号 | 规则描述 | 去重保护 |
|---------|---------|---------|
| R17-01 | 四个选项卡共享独立的输入状态，切换选项卡不清空其他选项卡的已有内容；用户切回之前的选项卡时可看到之前的输入和结果；**重复点击当前已激活的选项卡则忽略**，不做任何状态变更。 | — |
| R17-02 | "格式化" 和 "检查" 按钮在对应输入区域为空时显示为禁用态；输入区域有内容后自动变为可点击的蓝色状态。 | — |
| R17-03 | "Diff 对比" 按钮仅在两个输入面板均有内容时可点击；任一面板为空时按钮禁用。 | — |
| R17-04 | 格式化操作统一使用 4 空格缩进，指令块（`{}` 内部）逐级缩进，`server`、`location` 等块级指令前保留一个空行，注释行不调整缩进。 | — |
| R17-05 | 语法检查支持以下常见错误检测：缺少分号、未闭合的花括号、重复的指令、无效的指令参数、嵌套层级错误；错误提示精确到行号。 | — |
| R17-06 | Diff 对比采用行级对比算法，忽略行尾空白差异；同步滚动两个面板以确保用户可对比对应行。 | — |
| R17-07 | 格式化结果的右侧面板为只读模式，不支持直接编辑；用户需修改左侧输入面板的内容后重新格式化。 | — |
| R17-08 | 模板列表中的配置均为合法且可直接使用的 Nginx 配置片段，包含必要的注释说明关键参数的作用。 | — |
| R17-09 | 输入面板支持行号显示，语法检查和 Diff 对比的错误标注均以行号为定位依据。 | — |
| R17-10 | 语法高亮规则适用于格式化面板和检查面板：Nginx 指令关键字（`server`、`location`、`proxy_pass` 等）以蓝色显示，字符串参数以绿色显示，注释以灰色显示，花括号以加粗显示。 | — |
| R17-11 | 模板卡片采用网格布局，每行显示 2-3 个卡片（根据窗口宽度自适应），卡片包含模板名称、描述和适用场景标签。 | — |
| R17-12 | 所有操作按钮点击后显示 loading 状态（按钮文字变为旋转图标），操作完成后恢复为正常状态；若操作耗时超过 2 秒，显示进度提示。 | — |
