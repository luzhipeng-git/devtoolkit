# IX-19 JWT / 颜色转换 交互流程

> 页面文件：`19-codec.html`

## 界面状态定义

| 状态名称 | 触发条件 | 界面表现 |
|---------|---------|---------|
| 初始态 | 页面首次加载，默认显示 JWT 解码子工具 | 子工具选项卡（JWT 解码、HTML Entity、颜色转换）可见，当前选中 "JWT 解码" 选项卡以蓝色下划线标识；JWT 输入文本域为空，显示 placeholder "请粘贴 JWT Token…"；解码按钮为禁用态；三个解码片段区域（Header、Payload、Signature）均显示空状态提示 |
| 解码态 | 用户输入合法 JWT 并点击 "解码" 按钮 | JWT 输入文本域下方显示三个水平排列的片段卡片：**Header** 卡片以红色左边框标识，显示 base64 解码后的 JSON（算法、类型等字段）；**Payload** 卡片以紫色左边框标识，显示解码后的 JSON（含标准字段如 `iat`、`exp`、`sub` 等以高亮显示）；**Signature** 卡片以蓝色左边框标识，显示签名算法和原始 base64 字符串。每个卡片右上角有 "复制" 按钮。Payload 卡片下方显示 Token 状态信息（是否过期、过期时间） |
| 异常态 | 用户输入非法 JWT 格式（非三段式、base64 解码失败） | JWT 输入文本域边框变红，文本域下方显示红色错误提示（如 "无效的 JWT 格式：Token 必须由三部分组成，以 `.` 分隔" 或 "Header 部分 Base64 解码失败"）；三个片段区域清空并显示错误图标 |
| HTML Entity 初始态 | 用户切换到 "HTML Entity" 子工具 | 输入文本域为空，显示 placeholder "请输入需要编码/解码的文本…"；上方显示方向切换（编码 / 解码）单选按钮，默认选中 "编码"；输出区域为空；"执行" 按钮为禁用态 |
| HTML Entity 结果态 | 用户输入内容并点击 "执行" 按钮 | 输出区域显示编码后（如 `&lt;div&gt;`）或解码后（如 `<div>`）的文本；右上角显示 "复制" 按钮 |
| 颜色转换初始态 | 用户切换到 "颜色转换" 子工具 | 颜色选择器（color picker 控件）显示默认黑色；HEX 输入框显示 `#000000`，RGB 输入框显示 `R:0 G:0 B:0`，HSL 输入框显示 `H:0 S:0% L:0%`；颜色预览块显示黑色方块 |
| 颜色转换联动态 | 用户修改任一颜色值输入或使用颜色选择器 | 颜色预览块实时更新为当前颜色；其余格式的输入框自动联动更新为对应的等价值；若输入值超出有效范围（如 RGB 值大于 255），输入框边框变红并自动校正为最近有效值 |

## 操作流程

1. 页面加载后默认显示 "JWT 解码" 子工具，用户可通过顶部选项卡在三个子工具之间切换。
2. **JWT 解码流程**：
   - 用户在输入文本域粘贴 JWT Token 字符串。
   - 解码按钮在输入非空时变为可点击状态，用户点击 "解码" 按钮。
   - 系统按 `.` 分割 Token 为三段，分别对 Header 和 Payload 进行 Base64 解码。
   - 解码成功后，三个片段卡片并排显示，Header 卡片（红色左边框）显示算法信息，Payload 卡片（紫色左边框）显示声明数据，Signature 卡片（蓝色左边框）显示签名信息。
   - Payload 卡片下方显示 Token 有效期信息：若包含 `exp` 字段，计算并显示是否过期及过期时间；若不包含 `exp` 则显示 "无过期时间声明"。
   - 用户点击任一卡片的 "复制" 按钮，复制该片段的解码 JSON 内容。
3. **HTML Entity 编解码流程**：
   - 用户切换到 "HTML Entity" 选项卡。
   - 选择操作方向：编码（文本转 HTML Entity）或解码（HTML Entity 转文本）。
   - 在输入文本域输入内容，点击 "执行" 按钮。
   - 输出区域显示转换结果，用户可点击 "复制" 按钮复制结果。
4. **颜色转换流程**：
   - 用户切换到 "颜色转换" 选项卡。
   - 用户通过以下任一方式输入颜色：拖动颜色选择器、在 HEX 输入框输入十六进制值、在 RGB 输入框输入分量值、在 HSL 输入框输入分量值。
   - 系统实时将输入颜色转换为其他格式并更新所有输入框和颜色预览块。
   - 用户点击任意输入框旁的 "复制" 按钮复制该格式的颜色值。

## 交互规则

| 规则编号 | 规则描述 | 去重保护 |
|---------|---------|---------|
| R19-01 | 三个子工具（JWT 解码、HTML Entity、颜色转换）共享独立状态，切换选项卡不清空其他子工具的已有内容；**重复点击当前已激活的子工具选项卡则忽略**。 | — |
| R19-02 | JWT Token 输入时自动去除首尾空白和 `Bearer ` 前缀（若存在），确保解析不受多余字符影响。 | — |
| R19-03 | JWT 解码成功后，Header 和 Payload 的 JSON 内容自动格式化（2 空格缩进），字段按字母序排列，标准 JWT 字段（`iss`、`sub`、`aud`、`exp`、`nbf`、`iat`、`jti`）以加粗显示并在旁边标注中文名称说明。 | — |
| R19-04 | JWT 的 Signature 片段不进行 Base64 解码（因为可能包含不可打印字符），仅显示原始 Base64 字符串和签名算法名称（从 Header 的 `alg` 字段读取）。 | — |
| R19-05 | JWT Token 过期状态判定：若 Payload 包含 `exp` 字段且当前时间超过 `exp` 值，显示红色 "Token 已过期" 标签和过期时间；否则显示绿色 "Token 有效" 标签和剩余有效时间。 | — |
| R19-06 | JWT 解码为纯客户端操作，不验证签名有效性（无密钥），在 Payload 卡片下方以灰色文字标注 "注意：仅解码 Token 内容，未验证签名有效性"。 | — |
| R19-07 | HTML Entity 编码支持完整字符集，包括 `<`、`>`、`&`、`"`、`'` 以及中文等 Unicode 字符；解码支持命名实体（`&lt;`）和数字实体（`&#60;`、`&#x3C;`）。HTML Entity 方向选择（编码/解码）为单选按钮组，重复选择当前已选中的方向则忽略。 | — |
| R19-08 | 颜色转换输入验证：HEX 值须为 3 位或 6 位十六进制（可带 `#` 前缀）；RGB 各分量范围 0-255；HSL 中 H 范围 0-360，S 和 L 范围 0-100%。超出范围的值自动校正到最近有效值并显示黄色提示 "值已自动校正"。 | — |
| R19-09 | 颜色转换采用实时联动机制，任一输入框内容变化后即时更新其他输入框和预览块，无需点击按钮触发，防抖延迟为 150ms。 | — |
| R19-10 | 各子工具的复制按钮点击后短暂显示 "已复制" 反馈（按钮文字变色 2 秒），使用 `navigator.clipboard` API 实现。 | — |
