# IX-18 配置文件转换交互流程

> 页面文件：`18-config-converter.html`

## 界面状态定义

| 状态名称 | 触发条件 | 界面表现 |
|---------|---------|---------|
| 初始态 | 页面首次加载 | 方向选择器显示为 "JSON → YAML"（默认方向），源格式下拉框选中 JSON，目标格式下拉框选中 YAML，中间箭头图标指向右方；左侧源面板为空，显示 placeholder "请输入要转换的配置内容…"；右侧结果面板为空，显示空状态提示 "转换结果将显示在这里"；"转换" 按钮为禁用态；"自动检测" 按钮为可点击状态 |
| 转换态 | 用户输入内容后点击 "转换" 按钮，系统正在执行转换 | "转换" 按钮显示 loading 状态（旋转图标），左右面板内容保持不变，等待转换完成 |
| 结果态 | 转换成功完成 | 右侧面板显示转换后的配置内容，带有对应格式的语法高亮；面板顶部显示 "转换成功" 绿色提示（3 秒后自动消失）；结果面板右上角显示 "复制" 和 "下载" 按钮 |
| 异常态 | 源内容解析失败（格式错误） | 左侧源面板中错误行以红色底色标注，面板下方显示红色错误提示（包含行号和错误描述，如 "第 5 行：缺少逗号分隔符"）；右侧面板清空并显示错误图标和 "转换失败" 提示 |

## 操作流程

1. 用户通过顶部方向选择器选择源格式和目标格式，支持以下格式互转：JSON、YAML、TOML、INI、Properties、ENV。
2. 源格式下拉框位于左侧面板上方，目标格式下拉框位于右侧面板上方，两者之间显示一个可点击的箭头图标（`→`），点击箭头可交换源格式和目标格式。
3. 用户在左侧面板输入或粘贴待转换的配置内容。
4. 用户可点击源格式下拉框旁的 "自动检测" 按钮，系统分析输入内容并自动选择最可能的源格式，更新源格式下拉框的选中值。
5. 用户点击面板之间的 "转换" 按钮，系统解析左侧内容并将其转换为目标格式，结果显示在右侧面板。
6. 若转换成功，进入 **结果态**：右侧面板显示格式化的转换结果。
7. 若源内容存在语法错误，进入 **异常态**：标注错误行号并显示错误信息。
8. 用户点击右侧面板的 "复制" 按钮，将转换结果复制到系统剪贴板，显示 "已复制" 反馈提示。
9. 用户点击右侧面板的 "下载" 按钮，以目标格式的文件扩展名保存文件（如 `.json`、`.yaml`、`.toml` 等）。
10. 用户修改左侧面板内容或切换格式方向后，需重新点击 "转换" 按钮执行转换，右侧面板内容保持上次结果不变直到新转换完成。

## 交互规则

| 规则编号 | 规则描述 | 去重保护 |
|---------|---------|---------|
| R18-01 | "转换" 按钮在左侧面板为空时禁用；左侧面板有内容后自动变为可点击的蓝色状态。 | — |
| R18-02 | 源格式和目标格式不能选择相同的格式；若用户选择相同格式，目标格式自动切换为下一个可用格式并显示提示 "目标格式已自动调整"；**重复选择当前已选中的源格式或目标格式则忽略**。 | — |
| R18-03 | 方向交换（点击中间箭头）：交换源格式和目标格式的下拉框值，同时交换左右面板的内容——当前右侧的转换结果移至左侧作为新的源输入，右侧面板清空。 | 无需去重（交换按钮有明确意图） |
| R18-04 | 自动检测功能按以下优先级识别格式：JSON（以 `{` 或 `[` 开头）、YAML（含 `---` 或 `key:` 模式）、TOML（含 `[section]` 和 `key = value` 模式）、INI（含 `[section]`）、Properties（含 `key=value` 模式）、ENV（含 `KEY=VALUE` 模式）。无法识别时提示 "无法自动识别格式，请手动选择"。 | — |
| R18-05 | 转换错误精确到行号，错误信息包含：错误行号（如有）、错误类型（语法错误、类型不兼容等）、错误描述；对于多行格式（JSON、YAML），标注出错的行和列。 | — |
| R18-06 | 语法高亮根据目标格式应用对应规则：JSON（键为蓝色、字符串为绿色、数字为橙色、布尔/null 为紫色）；YAML（键为蓝色、字符串为绿色、注释为灰色）；TOML（表头为蓝色、键为绿色、字符串为橙色）；INI（节名为蓝色、键为绿色）；Properties（键为蓝色）；ENV（键为紫色）。 | — |
| R18-07 | 转换过程中保留原始数据的注释（仅 YAML、TOML、INI、Properties 支持注释）；JSON 和 ENV 格式不支持注释，转换时注释内容丢失并在结果面板顶部以黄色提示说明 "部分注释在目标格式中无法保留"。 | — |
| R18-08 | 格式下拉框选项中每个格式旁边显示对应的文件图标，便于视觉识别。 | — |
| R18-09 | 复制操作使用 `navigator.clipboard` API，复制成功后按钮文字短暂变为 "已复制" 并在 2 秒后恢复为 "复制"；若浏览器不支持剪贴板 API，回退至 `document.execCommand('copy')`。 | — |
| R18-10 | 下载文件名默认为 `converted.<ext>`，其中 `<ext>` 为目标格式的文件扩展名（json/yaml/toml/ini/properties/env）；用户可在下载对话框中修改文件名。 | — |
